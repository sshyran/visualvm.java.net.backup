This document describes VisualVM CVS layout and shourt instructions how to build VisualVM. There are two suites - one visualvm, which is the core tool and the plugins suite, which has plugins available on plugin center. The layout in CVS is

  • visualvm
    • www <- web pages
    • visualvm <- core tool suite
    • plugins <- for plugin center (jconsole, mbeans, glassfish)

To build visualvm core tool you need NetBeans platform and profiler binaries available here. Those binaries must be extracted to visualvm/visualvm directory. You can use ant run or ant build-zip to build or run VisualVM.

To build plugins suite, use ant build or ant run. This will automatically build zip distribution of core tool to visualvm/visualvm/dist/visualvm.zip and extract it to visualvm/plugins/visualvm directory. After that build of plugins suite continue as usually.
