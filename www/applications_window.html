Using the Applications Window

When you start VisualVM, the Applications window is visible in the left of the VisualVM window. The Applications window uses a tree structure to enable you to quickly view the applications that are running on the local JVM and connected remote hosts. The Applications window also displays core dumps and application snapshots.

Applications Window Nodes

The Application window has nodes and sub-nodes that you can expand to view running applications and saved files. For most nodes in the Applications window, you can view additional information and perform actions by right-clicking a node and choosing an item from the popup menu. The actions available in the popup menu vary according to the node.

The Applications window has the following primary nodes:

Local Node

The Local node displays the name and process IDs (PID) of Java applications that are running on the same local JVM as VisualVM. When you launch VisualVM and expand the local node, VisualVM automatically displays the currently running Java applications. VisualVM is always listed as one of the local applications. When a local Java application is launched, a node for that application appears under the Local node.

If you use VisualVM to take thread dumps, heap dumps or profiling snapshots of an application, they are displayed as sub-nodes below the application node. The application nodes under the Local node are updated dynamically. When an application starts, a node for the application is automatically added. When an application is terminated, the node and sub-nodes for the application are removed.

You can right-click a dump or snapshot sub-node to save it to your local system. You can create an application snapshot to collect and archive all the information collected about an application. To create an application snapshot, right-click the application node and choose Application Snapshot. Saved application snapshots are visible below the Snapshots node in the Applications window.

Remote Node

When VisualVM connects to a remote host, the remote host is listed as a node under the Remote node. When connected to a remote host, you can expand the node for the remote host to see the Java applications that are running on that host. When a Java application is launched on the remote host, a node for that application is displayed under the remote host node.

You need to explicitly add the location of remote hosts if you want VisualVM to be able to connect to the host. Once a remote host is added, VisualVM persists the address of the remote host and will make a connection to the host each time VisualVM is started. If you do not want VisualVM to connect to the host on startup, you should right-click the remote host node and choose Remove before quitting VisualVM.

Note: To retrieve and display information on applications running on the remote host, the jstatd utility needs to be running on the remote host. To start the jstatd utility, type jstatd on the command line. The jstatd utility is included as part of JDK 6. jstatd - Virtual Machine jstat Daemon

Snapshots

The Snapshots node lists application snapshots that are saved to your local system. You can right-click the Snapshots node to open save application snapshots.

An application snapshot can be saved to your local system and opened

Each application snapshot has the following items available in the popup menu.

  • Remove
  • Rename
