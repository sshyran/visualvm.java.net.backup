Applications Window

When you start VisualVM, the Applications window is visible in the left of the VisualVM window. The Applications window enables you to quickly view running local and remote applications. The Applications also displays core dumps and profiling snapshots saved to your local system.

You can perform actions on listed items by right-clicking the node in the Applications window and choosing an action from the popup menu.

Applications Window Nodes

The Applications window has the following nodes:

  • Local. Expand the Local node to view the applications running on the local JVM.
  • Remote. Expand the Remote node to view the connected remote hosts. Expand the node of a remote host to view the Java applications running on the remote host.
  • Core Dumps.
  • Snapshots. Expand the Snapshots node to view profiling snapshots saved to the local system.

Local Node

The Local node displays the Java applications that are running on the same local JVM as VisualVM. VisualVM is listed as one of the local applications. When a Java application is launched, a node for that application is displayed under the Local node. The Applications window also displays the process ID (PID) for the application.

Each local application has the following items available in the popup menu.

  • Open. Select this to open the tab for the application
  • Thread Dump. Select this to capture a thread dump for the application. When you capture a thread dump, the thread dump opens in the main window and a node for the thread dump appears under the application node.
  • Heap Dump. Select this to capture a heap dump for the application. When you capture a heap dump, the heap dump opens in the main window and a node for the heap dump appears under the application node.
  • Enable Heap Dump on OOME.

When you take a thread dump or heap dump or a local application, the dump appears under the application node.

Remote Node

The Remote node displays the remote hosts that VisualM is connected to. When you are connected to a remote host, you can expand the node for the remote host to see the Java applications that are running on that host. When a Java application is launched on the remote host, a node for that application is displayed under the Remote node. Double-click an application node to open the application tab.

Note. VisualVM can retrieve monitoring information on remote applications but it cannot profile remote applications.

Connecting to a Remote Host
To add a remote host, right-click the Remote node, choose Add Remote Host and type the host name or IP address in the Add Remote Host dialog box.

Note: To retrieve and display information on applications running on the remote host, the jstatd utility needs to be running on the remote host. To start the jstatd utility, type jstatd on the command line. The jstatd utility is included as part of JDK 6. jstatd - Virtual Machine jstat Daemon
