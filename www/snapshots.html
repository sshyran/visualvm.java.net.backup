Taking Snapshots

VisualVM enables you to take snapshots of application data and save the snapshots to your local system.

You can take and save the following types of snapshots:

Profiler Snapshots

Profiler snapshots capture profiling data at the moment the snapshot is taken

  • Memory snapshots. A memory snapshot captures profiling data on allocated objects. You can take a memory snapshot when you are using the profiler to analyze memory usage. You can compare memory snapshots session on a local application.
  • CPU snapshots. You can take a CPU snapshot when you are using the profiler to application performance.

Application Snapshots

An application snapshot is an archive of profiler snapshots of an application. You can take an application snapshot if you have memory or CPU snapshots for the application. - These are saved under the Snapshots node in the Applications window. Application snapshots contains heap and thread dumps. You can take an application snapshot of a local application if you have already created heap or thread dumps.

When you take a memory or CPU snapshot, the snapshot opens in the application's tab in the main window.

The Snapshots node lists profiling snapshots that are saved to your local system.

To take a snapshot during a profiling session, do one of the following:

  • Click the Take Snapshot of Collected Results button in the Profiler tab
  • Right-click the application node and choose Profiler Snapshot.

The snapshot toolbar enables you to do the following:

  • Save Snapshot to Custom File
  • Save Current View to Image
  • Find in Results
  • Find Next Occurence - find next occurence of search term
  • Find Previous Occurence -find previous occurence of search term
