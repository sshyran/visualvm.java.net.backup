Monitoring an Application

VisualVM presents data for local and remote applications in a tab specific for that application. When you open an application in VisualVM, each application opens in a new tab in the main window. You can have multiple application tabs open.

Each application tab contains sub-tabs that display different types of information about the application.

The Monitor tab displays real-time, high-level data about the memory heap thread activity and classes loaded in the VM.

The Monitor tab contains the following sub-tabs:

  • Heap. Displays the total heap size and how much of the heap is used currently heap. These are the figures that a Java application can obtain using the java.lang.Runtime.totalMemory() and java.lang.Runtime.freeMemory() calls.
  • PermGen. The permanent generation is the area of the heap where class and method objects are stored. If an application loads a very large number of classes, then the size of the permanent generation might need to be increased using the -XX:MaxPermSize option. The PermGen graph displays information retrieved using the jmap utility. The PermGen graph displays the relationship between the used permgen space and the total size of the permgen space.
  • Classes Displays information about class loading.
  • Threads. Displays the current number of live and daemon threads in the application's JVM.

When you are monitoring a local application, you can perform the following actions from within VisualVM.

  • Perform GC Click this to perform garbage collection immediately.
  • Heap Dump. Click this to take a heap dump. When you take a heap dump, a tab displaying the heap dump opens in the application tab. A node for the heap dump appears under the node for the application in the Applications window.
