Troubleshooting guide for VisualVM Preview 1


VisualVM Application Window Is Empty, Exception Is Thrown

Description: Main window of VisualVM is empty, java.lang.ClassNotFoundException: sun.jvmstat.monitor.MonitorException may be presented to the user and is logged in the logfile.

Resolution:  This happens when VisualVM runs on JRE instead of JDK. To define which JDK will be used for running VisualVM, follow the steps in Getting Started section of the Introduction to VisualVM document.


No Running Local Applications Detected

Description: There are no running applications shown in the Applications window on the left side (including the VisualVM itself).

Resolution:  This can happen on Windows systems, it's a known problem related to the jps JDK tool used by VisualVM to detect running Java applications. There are two main root causes: broken access rights to temporary directory on a NTFS disk - see this thread for details and fix or having temporary directory on a FAT disk - see this bug detail for details and fix.


No Running Remote Applications Detected

Description: There are no running applications shown for remote host in the Applications window on the left side.

Resolution:  Make sure that the jstatd utility is running on the remote host you want to monitor. This is a requirement for VisualVM to be able to discover and access remote Java applications. More information about jstatd can be found in Remote Applications section of the Introduction to VisualVM document.


Profiling Does Not Work

Description: Profiling doesn't work for applications running on JDK 6.0.

Resolution:  There is a known problem with dynamic attach which is used for profiling, it may cause target JVM to crash when class sharing is enabled. To start an application without class sharing, provide -Xshare:off argument to the java command starting the app. More information can be found in this and this bug detail.


Profiling Does Not Work on Mac OS X

Description: Profiling doesn't work for applications running on Developer preview of JDK 6.0 on Mac OS X.

Resolution:  Unfortunately profiling does not work with Developer preview of JDK 6.0, due to bug in the JDK. However you can still take thread and heap dumps of applications running on JDK 6.0 on Mac OS X.
