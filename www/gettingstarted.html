Getting Started with VisualVM

VisualVM provides detailed information about Java applications while they are running on the Java Virutal Machine (JVM). VisualVM's graphical user interface enables you to quickly and easily see information about multiple Java applications.

This getting started guide should help you quickly get up and running with the VisualVM. This guide will demonstrate how to install VisualVM and extend the tool by adding the JMX-MBean plugin from the VisualVM update center. In this guide you will launch VisualVM and view the details of an application running on the local JVM.

This guide will also demonstrate how to connect to an application running on a remote JVM.

Setting Up

Installing VisualVM

  1. Download and unzip the VisualVM installer.
  2. Navigate to the bin directory of the VisualVM installation directory and launch the application.

When you launch VisualVM you see the main window on the application.

Installing VisualVM Plugins

You can extend the functionality of VisualVM by installing plugins available from the VisualVM update center. For example, installing the VisualVM-MBeans plugin adds a new MBeans tab to the application tab that is visible if the application utilizes JMX management.

To install a VisualVM plugin:

  1. Choose Tools > Plugins from the main menu.
  2. In the Available Plugins tab, select the Install checkbox for the plugin. Click Install.
  3. Step through and complete the plugin installer.

In this screenshot the VisualVM-MBeans plugin is selected. The plugin adds an MBeans that is visible when an application uses JMX management. The tab enables you to easily view JMX data for local applications.
Note: You will need to restart VisualVM for the MBeans tab to appear.

Viewing Application Data

By default, the Applications window is displayed in the left pane of the main window when VisualVM is launched. The Applications window enables you to quickly view data on running Java applications. Local applications are listed below the Local node. Java applications running on connected remote hosts are listed below the Remote node. When you double-click the application name in the Applications window, a tab for that application opens in the main window. Each application tab contains tabs that you use to explore the application data in greater detail.
Note. VisualVM is a Java application and will always be one of the applications listed under the Local node.

Profiling an Application

VisualVM includes a profiler that enables you to profile applications running on your local JVM. You access the profiling controls in the Profiler tab of the application. You can analyze memory usage and CPU performance.
Note. To profile an application running on JDK 6, you need to turn off class sharing for the application otherwise the application may crash. To turn off class sharing, start the application with -Xshare:off argument.

  1. Start VisualVM (if not already started)
  2. Launch a local Java application with the -Xshare:off argument.
  3. Expand the Local node in the Applications window and double-click the name of the Java application.
  4. Click the Profiler tab and then click the Memory button.

    When you click Memory, VisualVM connects to the application and displays the profiling data.

Connecting to a Remote Host

VisualVM enables you to easily monitor applications running on remote hosts and view general data about the remote system. You cannot profile applications running on a remote host.

You can also add remote hosts to view details about applications running on remote JVMs. Remote applications are listed below the Remote node in the Applications window.

To retrieve data from a remote application, the jstatd utility needs to be running on the remote JVM. For more information on how to start jstatd, see jstatd - Virtual Machine jstat Daemon

  1. Right-click Remote in the Applications window and choose Add Remote Host.
  2. In the Add Remote Host dialog box, type the host name or IP address of the remote machine.
  3. (Optional) Type a display name for the remote host. This name is displayed in the Applications window. If no display name is entered, the host name is used to identify the remote host in the Applications window.
  4. Click OK.

When you click OK, a node for the remote host appears under the Remote node. When you expand the remote host node you can see the Java applications that are running on the remote host.

You can double-click on the name of the remote applications to open the application tab in VisualVM.
